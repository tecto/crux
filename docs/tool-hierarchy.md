# Tool Resolution Hierarchy

## Five-Tier System

Tools are organized in tiers by reliability and determinism. The system always prefers higher (lower-numbered) tiers.

### Tier 0: LSP (Language Server Protocol)
- Deterministic code intelligence
- Jump to definition, find references, type information
- Configured: pyright (Python), elixir-ls (Elixir)
- Lazy-loaded (no cost until used)

### Tier 1: Custom Tools
- Schema-validated inputs (Zod)
- Atomic, deterministic operations
- `run_script`, `lookup_knowledge`, `suggest_handoff`, `project_context`, `promote_script`, `list_scripts`, `manage_models`

### Tier 2: MCP Servers
- Protocol-enforced external access
- GitHub MCP, filesystem MCP, browser MCP
- High reliability through protocol standardization

### Tier 3: Library Scripts
- Pre-tested, versioned in `.opencode/scripts/lib/`
- Proven safe through previous execution
- `backup_database.sh`, `run_tests.sh`

### Tier 4: New Scripts
- Generated by model, validated through five-gate pipeline
- One-off operations and experimental automation

### Tier 5: Raw Bash
- Fallback for operations that don't fit higher tiers
- Read-only only (cannot modify files)
- `ls`, `grep`, `find` (information gathering)

## Enforcement

The `tool-enforcer.js` plugin intercepts every tool call and:
1. Checks mode-specific access restrictions
2. Blocks write tools in read-only modes
3. Tracks tier usage per mode for optimization analytics
4. Flags high Tier 5 usage for custom tool creation
